# 思维导图文本格式约束

## 当前范围

第一版文本格式只作用于“选中主题整体文本”，不作用于主题内部的局部选区。

支持字段：

- `fontWeight`: 加粗 / 常规
- `fontStyle`: 斜体 / 常规
- `textDecoration`: 下划线 / 无
- `color`: 文字颜色
- `fontSize`: 字号

## 数据入口

- UI 组件：`MindMapTextFormatToolbar`
- 画布组件窄接口：`MindMapCanvasHandle.applyTextFormat`
- 编辑器适配层：`simpleMindMapAdapter.applySelectedTextFormat`
- 持久化：沿用导图快照保存，不新增独立格式表。

## 稳定性原则

- 格式字段必须写入 simple-mind-map 原生节点样式字段。
- 禁止直接改 DOM 样式，因为 DOM 改动无法稳定进入快照、撤销、导出和重开流程。
- 禁止在 UI 组件中访问 simple-mind-map 实例。
- 禁止把文本格式作为目录字段保存，目录只读展示导图树。
- 禁止按节点标题匹配格式目标，必须使用当前编辑器选中节点。

## 清除格式

清除格式时字段写入 `undefined`，让主题层级样式重新接管。

不要把清除格式实现成固定颜色、固定字号或固定字重，否则会破坏根节点、二级节点、普通节点各自的主题样式。

## 暂不接入

暂不接入富文本局部选区格式，包括：

- 单个主题内局部文字加粗
- 单个主题内局部文字多色
- 局部链接、局部背景色

这些能力需要先启用并验证 simple-mind-map 的 RichText 插件，单独设计保存、导出、撤销、粘贴兼容策略后再做。
