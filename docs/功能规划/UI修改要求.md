# UI 修改要求

## 1. 目的

本文用于约束 AIstudy 后续 UI 与 CSS 修改。任何界面改动都必须优先保证正式产品感，不允许把开发说明、实现说明、测试说明或操作教程直接放到用户界面上。

## 2. 禁止出现在 UI 上的内容

以下内容只能写在文档、注释、日志或开发任务里，禁止显示在应用界面中：

- 开发状态说明：例如「待开发」「下一步接入」「功能未完成」「预留区域」。
- 实现细节说明：例如「保存到 userData 文件」「写入 localStorage」「读取 courses.json」。
- 测试或验收说明：例如「用于验证」「持久化验证」「测试课程」。
- 操作教程式文案：例如「点击右上角加号创建第一门课程」「选择课程后显示思维导图」。
- 解释布局用途的文案：例如「这里会显示」「挂载区域」「关联文档区域」。
- 英文占位或框架默认文案，除非是专有名词或用户明确要求。

## 3. 允许出现在 UI 上的内容

界面文案必须短、稳、像正式产品：

- 页面名称：知识库、工作台、节点详情。
- 状态短语：暂无课程、未选择课程、暂无图谱、暂无节点。
- 动作名称：新建、保存、取消、重命名、删除、搜索。
- 用户输入内容：课程名称、课程描述、节点标题、文档正文。
- 必要错误提示：保存失败、读取失败、删除失败。

## 4. 空状态规范

空状态只表达当前状态，不解释功能路线。

推荐：

- 暂无课程
- 未选择课程
- 暂无图谱
- 暂无节点

不推荐：

- 点击右上角加号创建第一门课程。
- 课程已创建，下一步接入节点和思维导图编辑。
- 选择思维导图节点后显示关联文档。
- 接入课程数据后会显示在这里。

## 5. 课程列表规范

- 课程卡片只显示用户真实输入的课程名称和描述。
- 如果用户没有填写描述，不显示描述占位文案。
- 禁止在课程卡片中显示开发验证文案、存储路径、数据层说明。
- 开发或测试课程必须在交付前清理，不得留在用户数据中。

## 6. CSS 与布局规范

- UI 不用大段说明文字撑布局。
- 信息密度以桌面工具为准，保持紧凑、清晰、可扫描。
- 按钮只展示动作，不用旁边文字解释按钮用途。
- 禁用态按钮必须视觉弱化，但不要出现解释性说明。
- 文字不得压线、重叠、截断到不可识别；必要时用省略号或增加容器高度。
- 不用教程式卡片、营销式说明块或开发占位卡片。

## 7. 交付检查清单

每次 UI/CSS 修改后检查：

- 页面上没有「待开发」「下一步」「验证」「userData」「localStorage」「预留」「接入」等开发说明词。
- 页面上没有示例课程、测试课程或虚拟课程。
- 空状态只保留短状态文案。
- 所有文案为中文。
- `npm run build` 通过。
- 打包后的本地 exe 与源码显示一致。
