# 系统核心逻辑冻结契约

## 目标

从本版本开始，系统只允许围绕「课程 + 思维导图 + Word 文档」这条主链路扩展。后续新增 AI、导出、复习卡、素材库、端口管理等功能时，不能改变这条链路的数据归属和身份规则。

## 唯一身份链

1. `course_id` 是课程容器身份。
2. `mind_map_id` 是课程下唯一导图身份。
3. `node_id` 是导图节点稳定身份，来自 `simple-mind-map` 节点 `data.uid`。
4. Word 详情文档只允许通过 `course_id + mind_map_id + node_id` 绑定。
5. 标题、目录路径、UI 选中状态、节点显示顺序都不能作为跨功能绑定键。

## 思维导图契约

- 导图编辑器固定为 `simple-mind-map`。
- 导图快照协议固定为 `schemaVersion: 1` 和 `editor: "simple-mind-map"`。
- 节点树 `root.children` 是目录父子级的唯一来源。
- 节点标题允许重复，重命名不得改变节点 ID。
- 目录只允许由 `buildMindMapOutline(root)` 生成。
- 目录折叠只属于 UI 状态，不落库，不反写导图树。
- 删除节点后，节点投影软删除，同时相关 Word 文档先软删除。

## Word 文档契约

- Word 编辑器固定为 `@hufe921/canvas-editor`。
- Word 快照协议固定为 `schemaVersion: 1` 和 `editor: "aistudy-word"`。
- `knowledge_documents` 只保存当前文档指针和轻量元数据。
- `knowledge_document_snapshots` 保存实际编辑器内容快照。
- 打开课程时不得加载整门课所有 Word 内容。
- 切换节点或切换编辑器模式前必须先保存当前未落库内容。
- 保存 Word 前必须校验对应 `mind_map_nodes` 节点存在。

## 存储与膨胀控制

- 不用硬性限制运行时内存作为主要方案。
- 通过按需加载、编辑器卸载、快照 hash 复用、快照保留数量、二进制资产外置来控制膨胀。
- 导图快照当前保留 12 份。
- Word 快照当前保留 16 份。
- 图片、附件、二进制内容后续必须进入资产表，不能以内联 base64 塞入导图或 Word JSON。
- 资产主表固定以 `sha256` 去重；引用关系固定写入资产关联表，不反向塞回导图或 Word 快照。

## 代码边界

- Electron main 进程拥有 MySQL、文件、更新、Chrome 端口和 IPC。
- React renderer 只拥有 UI 状态和编辑器挂载。
- Renderer 禁止直接访问 MySQL。
- `simple-mind-map` 的私有实例只能出现在 `simpleMindMapAdapter.ts`。
- `canvas-editor` 的私有实例只能出现在 `canvasEditorAdapter.ts`。
- 业务组件只能通过适配器暴露的稳定 handle 调编辑器能力。
- 新功能必须优先按模块落地；模块目录必须包含独立 `README.md`，说明范围、边界、流程和扩展规则。
- 新功能开发前必须先查找 GitHub 或开源项目的成熟做法，能复用设计经验时不得闭门重造。

## 修改准入

后续任何功能如果要改以下内容，必须先更新本契约，再改代码：

- 节点 ID 生成和归一化规则。
- 目录父子级生成规则。
- Word 文档绑定键。
- 导图或 Word 快照协议字段。
- MySQL 当前指针表和快照表职责。
- 编辑器适配层暴露的 handle 方法。

## 建议写死的底层规则

这些规则属于系统地基，不建议做成普通配置项，也不建议让 UI 或业务功能临时覆盖：

- `schemaVersion`、`editor`、`editorVersion` 协议字段必须固定在适配层和存储层，不允许由组件临时拼接。
- Word 文档绑定键固定为 `course_id + mind_map_id + node_id`，任何导入、AI 生成、导出、复习卡功能都只能沿用这个键。
- 「未分区」固定用 `section_id = NULL` 表达，不在数据库里创建一条假分区记录。
- 课程和分区排序固定为 `sort_order` 优先、`updated_at` 兜底；拖拽功能只能补重排命令，不能改排序语义。
- 课程排序只能通过 `courses:reorder`，分区排序只能通过 `course-sections:reorder`，不得由渲染层直接重写整份课程列表。
- `courses:save` 固定为旧接口兼容入口，新功能只允许走命令式 IPC。
- MySQL 失败时 pending 队列只允许记录课程和分区索引操作，禁止写入导图快照、Word 快照、附件、AI 结果。
- 报错日志固定为“前端人话 + 后端技术详情”双层结构，用户界面不得显示 SQL、堆栈、文件路径或内部异常原文。
- 新模块必须接入统一错误码目录，禁止直接把原始异常透传到 UI。
- 新模块不得跳过开源参考调研；调研结论应进入模块 README 或功能规划文档。
- 空描述、空调试信息、占位解释文案不得外显；列表项只显示用户真实输入内容。
- 导图快照、Word 快照保留数量和单条体积上限应由主进程存储服务统一控制，组件不得自行绕过。
- 二进制资产必须外置到资产表或文件存储，不允许作为 base64 长文本进入导图或 Word JSON。
- 资产关联里的 `document_id` 空值固定使用空字符串，不使用数据库 `NULL`，避免唯一索引失效导致重复引用。
