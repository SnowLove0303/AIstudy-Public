# 开箱即用与外部接入规划

## 目标

同一套 AIstudy 安装包搬到新电脑或新系统时，核心学习功能必须能先跑起来。外部接入只能增强能力，不能阻断知识库、导图、文档这条主链路。

## 能力分级

### 必须可本机运行

这些能力属于产品地基，不能依赖联网、登录或外部服务才能打开：

- 知识库列表。
- 课程分区和排序。
- 思维导图编辑。
- Word 详情文档编辑。
- 本机镜像、pending 队列和异常隔离。
- 资产目录和资产索引规则。

### 可选外部接入

这些能力可以失败，但失败时只能显示人话状态，不能影响核心编辑：

- MySQL 正式存储和同步。
- Chrome 固定端口。
- 豆包网页会话。
- ChatGPT 网页会话。
- GitHub Release 更新。
- 后续导入器、导出器、AI 生成模块。

## 数据根目录

运行时数据统一进入 `AIstudyPublicData`，后续禁止新增散落路径。

优先级：

1. `AISTUDY_PUBLIC_DATA_ROOT` 环境变量。
2. 开发态：项目 `.runtime` 目录。
3. 非 C 盘安装：`AIstudyPublic.exe` 同级 `AIstudyPublicData`。
4. 有 F 盘：`F:\AIstudyPublicData`。
5. 兜底：系统 `userData\AIstudyPublicData`。

目录结构：

```text
AIstudyPublicData/
  config/
    mysql.config.json
  state/
    courses.json
    course-pending-operations.json
  runtime/
    chrome-ports.json
    chrome-profiles/
  assets/
  updates/
  backups/
  logs/
```

旧版 `userData` 下的 `courses.json` 和 `course-pending-operations.json` 仍允许读取，避免升级后旧数据消失；新写入统一落到 `AIstudyPublicData/state`。

## 外部接入现状

### 设置页环境检查

设置里提供独立的“环境检查”页，用户可以一键检查新机器运行状态。检查结果只展示人话状态和处理建议，不展示 SQL、文件路径、堆栈或网页异常原文。

当前检查项：

- 数据目录是否能创建、读取和写入。
- 本机课程镜像和 pending 队列是否可读取。
- MySQL 是否可连接，错误日志表是否可用。
- Chrome 是否可找到。
- 豆包、ChatGPT 固定端口是否可连接、是否已登录。
- GitHub 更新服务是否可访问。

新增外部模块时，必须把自己的依赖检查接入这个页面，并声明失败后是否影响核心学习链路。

### MySQL

当前 MySQL 是正式数据源，本地 JSON 是轻量镜像和失败兜底。为了真正开箱即用，后续需要补本机默认数据库模式。MySQL 应成为高级同步模式，而不是启动前置条件。

当前要求：

- MySQL 失败不能阻断课程侧栏。
- 技术细节进入报错日志。
- pending 队列只记录课程和分区索引命令，不写导图、Word、资产、AI 结果。

### Chrome 端口

当前固定端口：

- 豆包：`9224`
- ChatGPT：`9230`

Chrome profile 进入 `AIstudyPublicData/runtime/chrome-profiles`。目标机器没有 Chrome、端口被占用、用户未登录时，只影响 AI 能力，不影响学习内容编辑。

### AI 网页会话

豆包和 ChatGPT 当前走网页会话，不走 API Key。它依赖页面结构、登录状态和网络环境。后续如果要产品化复用，应封装为可选 AI Provider。

### GitHub 更新

更新检测依赖 GitHub Releases。离线或内网失败时只影响更新页，不能影响核心工作区。安装包下载进入 `AIstudyPublicData/updates`。

## 后续必须补的地基

1. 本机默认数据库 Provider。
2. `StorageProvider` 接口：本机库和 MySQL 共享同一业务命令。
3. 本机默认数据库 Provider 接入环境检查，明确告诉用户当前是“本机模式”还是“MySQL 同步模式”。
4. 资产路径相对化：数据库只保存相对 asset key，不保存绝对路径。
5. 外部能力注册表：每个模块声明依赖、降级策略、错误码、是否可重试。

## 写死规则

- 核心学习链路不得依赖外部登录状态。
- 数据路径不得新增散落的 `userData` 文件。
- 资产不得以内联 base64 长文本进入导图或 Word JSON。
- 外部接入失败不得把 SQL、路径、堆栈、网页异常原文展示给用户。
- 新外部模块必须先接错误码体系，再接 UI。
