# AIstudy 功能开发需求

## 1. 文档目的

本文用于记录 AIstudy 从当前静态桌面壳进入功能开发阶段的需求范围。后续开发、排期、验收、复用需求时，以本文作为基础版本。

当前应用已经具备正式级桌面窗口、中文界面、固定应用图标和基础三栏布局，但页面上的交互入口尚未接入业务逻辑。现阶段原则是：不展示虚拟数据，不保留无对应页面的导航入口，未接入功能的按钮必须保持禁用或改为非交互展示。

## 2. 当前页面范围

当前保留唯一页面：知识库。

页面结构：

- 左侧窄导航：仅保留当前页面「知识库」标识。
- 左栏：课程列表、课程搜索、新建课程入口。
- 中栏：学习工作台、思维导图挂载区域、工作台工具入口。
- 右栏：节点详情、关联文档区域、详情收起入口。

## 3. 当前交互现状

当前所有业务按钮均未接入功能，必须按以下状态处理：

| 区域 | 入口 | 当前状态 | 后续目标 |
| --- | --- | --- | --- |
| 左侧导航 | 知识库 | 当前页标识，不跳转 | 后续新增页面时再扩展导航 |
| 左栏 | 新建课程 | 禁用 | 打开新建课程弹窗并保存课程 |
| 左栏 | 搜索课程或节点 | 静态展示 | 支持课程名、节点名、文档内容检索 |
| 中栏 | 思维导图 | 禁用 | 切换或聚焦思维导图视图 |
| 中栏 | 文档视图 | 禁用 | 切换到当前节点关联文档 |
| 中栏 | 更多 | 禁用 | 打开导入、导出、删除、设置等菜单 |
| 右栏 | 收起详情 | 禁用 | 折叠或展开右侧详情栏 |

## 4. 功能开发目标

### 4.1 课程管理（基础版已实现）

目标：用户可以创建、查看、选择、重命名、删除课程。

需求：

- 点击「新建课程」打开弹窗。
- 新建课程至少包含课程名称、描述、创建时间、更新时间。
- 课程保存后出现在左侧课程列表。
- 选择课程后，中栏标题从「未选择课程」变为课程名称。
- 删除课程前必须二次确认。
- 无课程时继续显示「暂无课程」空状态。

验收标准：

- 新建课程后刷新应用仍可看到课程。
- 课程列表不出现任何演示或虚拟课程。
- 未选择课程时，思维导图和详情栏保持空状态。

实现状态：

- 已实现新建、选择、搜索、重命名、删除课程。
- 已实现 Notion 式左侧树状分区的新建、重命名、折叠、删除，以及课程在分区和「未分区」之间移动。
- 已接入 Electron 主进程 MySQL 数据层，课程分区写入 `knowledge_sections`，课程数据写入专用表 `course_management_courses`。
- 课程与分区表连接、建库、建表规则详见 `docs/功能规划/MySQL课程管理表.md`。
- 后续仍需在课程删除时联动清理节点、边和文档。

### 4.2 本地数据持久化

目标：先实现离线可用的本地数据层，为后续同步或数据库迁移留接口。

需求：

- 保存课程、节点、节点关系、文档内容、最近打开记录。
- 数据模型必须区分课程、图谱节点、文档三类实体。
- 所有写入操作要有失败提示，不允许静默失败。
- 数据文件或数据库位置需要固定，并在文档中说明。

建议模型：

- Course：课程。
- MindMapNode：思维导图节点。
- MindMapEdge：节点关系。
- Document：节点关联文档。
- AppSettings：用户偏好和布局状态。

验收标准：

- 关闭应用再打开，数据不丢失。
- 数据为空时展示真实空状态。
- 损坏或读取失败时给出中文错误提示。

### 4.3 思维导图编辑器

目标：中栏成为课程知识图谱的核心编辑区域。

需求：

- 选择课程后加载该课程的思维导图。
- 支持新增节点、编辑节点标题、删除节点。
- 支持建立父子关系或连线关系。
- 支持拖拽节点调整布局。
- 支持选中节点，并把节点详情同步到右栏。
- 支持撤销、重做、快捷键和自动保存。

验收标准：

- 新建节点后能保存并恢复。
- 选中节点时右栏展示对应标题和文档摘要。
- 删除节点会同步删除或处理关联边，并有确认提示。

### 4.4 节点详情与文档

目标：右栏用于展示和编辑当前节点的学习资料。

需求：

- 选中节点后展示节点名称、备注、关联文档。
- 支持为节点创建或编辑文档。
- 支持 Markdown 基础编辑能力。
- 支持文档自动保存和保存状态提示。
- 支持节点无文档时的空状态和创建入口。

验收标准：

- 切换节点时右栏内容同步变化。
- 编辑文档后重新打开应用仍能恢复。
- 未选中节点时不展示上一次节点的旧内容。

### 4.5 搜索

目标：搜索框从静态展示变为可用检索入口。

需求：

- 支持按课程名称搜索。
- 选择课程后支持按节点标题搜索。
- 后续支持文档正文检索。
- 搜索结果为空时展示中文空状态。
- 搜索输入清空后恢复默认列表。

验收标准：

- 输入关键字后列表实时过滤。
- 搜索不会生成虚拟结果。
- 中文、英文、数字关键词均可检索。

### 4.6 工作台工具栏

目标：中栏右上角工具按钮具备明确功能。

需求：

- 思维导图按钮：切换或聚焦图谱视图。
- 文档视图按钮：切换到当前节点文档视图。
- 更多按钮：打开菜单，包含导入、导出、课程设置、删除课程。
- 未选择课程时工具栏保持禁用。
- 选择课程后按可用条件启用按钮。

验收标准：

- 禁用状态下按钮不可触发。
- 可用状态下点击有明确反馈。
- 更多菜单不会遮挡窗口边界。

### 4.7 详情栏折叠

目标：右栏可以被收起，为思维导图编辑留出空间。

需求：

- 点击「收起详情」隐藏右栏。
- 收起后中栏自动占用更多宽度。
- 提供恢复详情栏的入口。
- 记住用户上一次的详情栏展开状态。

验收标准：

- 折叠和展开不导致内容错位。
- 小屏幕下仍符合响应式布局。
- 重启应用后恢复上一次布局偏好。

### 4.8 导入导出

目标：为课程资料迁移和备份提供能力。

需求：

- 支持导出单个课程为本地文件。
- 支持导入课程文件并合并到课程列表。
- 导入前校验文件格式。
- 导入失败时展示中文错误信息。

验收标准：

- 导出的课程可以在当前应用重新导入。
- 导入不会覆盖已有课程，除非用户明确确认。
- 导入后的节点和文档关系完整。

## 5. 非功能要求

- 所有界面文案必须为中文。
- 不允许使用虚拟课程、虚拟节点、虚拟文档作为默认数据。
- 未完成的功能入口必须禁用或隐藏。
- 桌面窗口必须使用原生窗口框架，关闭、最小化、最大化可用。
- 应用图标固定使用 `build/icon.ico`，后续打包不得回退默认 Electron 图标。
- 页面在 1080px 最小宽度下不得出现文字互相遮挡。
- 错误、空状态、保存状态必须有清晰中文反馈。

## 6. 推荐开发顺序

1. 数据模型与本地持久化。
2. 课程新建、选择、删除。
3. 左侧课程列表和搜索。
4. 思维导图节点 CRUD。
5. 节点详情和文档编辑。
6. 工具栏菜单、导入导出。
7. 详情栏折叠和布局状态保存。
8. 打包验收和桌面快捷方式验证。

## 7. 阶段验收清单

- 应用启动后没有乱码。
- 左侧导航只显示当前页面入口。
- 页面没有虚拟数据。
- 禁用按钮不会触发任何动作。
- 已接入功能的按钮可点击、有反馈、能持久化。
- 关闭应用后重新打开，课程、节点、文档、布局状态不丢失。
- `npm run build` 通过。
- 打包后 exe 能打开和正常关闭。

## 8. 复用说明

后续新增功能需求时，优先追加到本文对应章节。如果需求已经实现，应在对应条目后标注实现状态、关联文件和验收方式，避免需求散落在聊天记录或临时说明中。
