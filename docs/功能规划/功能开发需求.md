# AIstudy 功能开发需求

## 1. 文档目的

本文用于记录 AIstudy 从当前桌面知识库进入持续功能完善阶段的需求范围。后续开发、排期、验收、复用需求时，以本文作为基础版本。

当前应用已经具备正式级桌面窗口、中文界面、固定应用图标、知识库工作台、思维导图编辑、文档编辑、导入器、设置、更新管理、报错日志和 MCP 外部接入能力。现阶段原则是：不展示虚拟数据，不保留无对应页面的导航入口，未接入功能的按钮必须保持禁用或改为非交互展示。

## 1.1 当前实现状态（0.1.61）

- 桌面端采用 Electron + React + TypeScript，主进程负责窗口、数据目录、MySQL、本地资源、导入导出、更新和 MCP 服务。
- 知识库支持知识体系、分区、课程的创建、重命名、移动、删除、搜索和持久化。
- 思维导图以 `simple-mind-map` 为核心，支持节点编辑、关系维护、自动保存、导出、目录同步、右键文字排版浮层和快捷键驱动的分支排版。
- 思维导图默认关闭自由节点定位，快照归一化时清理 `customLeft/customTop`，避免结构化布局出现分支穿线。
- 文档编辑以 `@hufe921/canvas-editor` 为核心，支持节点关联文档、分页阅读、基础排版、导入、自动保存、宽纸张纵向阅读流和页面尺寸修正。
- AI 聊天助手通过固定 Chrome 调试端口复用已登录的豆包/ChatGPT 页面，支持自动解析本机 Chrome 启动器；ChatGPT 会先写入网页输入框并保持焦点，再通过 CDP Enter 键发送，并按网页回合顺序读取本次回复。主进程 CDP 通讯使用 Node `ws` 客户端并安全解码 Buffer 消息。
- 设置页包含环境检查、MCP 控制台、更新管理、报错日志、快捷键设置等管理入口。
- MCP 已作为独立模块接入，支持全库读取、导图读取、节点搜索、导图编辑、健康检测、知识库定位、接入配置生成、内网访问、外部调用监控和远程权限开关。
- 内网访问基于 Tailscale，开启后可生成 MCP URL、API URL 和 Authorization 三行连接信息，供其他设备的 Codex 或 Claude Code 使用。
- 知识库左栏和右侧目录栏均支持折叠，折叠后只扩大中间画布，不卸载当前课程、导图或文档。

## 2. 当前页面范围

当前保留四类主要页面或模块入口：知识库、端口/浏览器管理、AI 助手、设置。

页面结构：

- 左侧窄导航：知识库、浏览器/端口、AI 助手、设置。
- 知识库左栏：课程搜索、知识体系、分区、课程列表、新建入口、分区和课程操作。
- 中间工作台：思维导图、文档视图、顶部工具栏、状态栏、画布滚动区域。
- 右侧目录栏：当前导图目录和节点层级导航。
- 设置弹窗：环境检查、MCP 控制台、更新管理、报错日志、快捷键设置等。

## 4. 功能开发目标

### 4.1 课程管理（基础版已实现）

目标：用户可以创建、查看、选择、重命名、删除课程。

需求：

- 点击「新建课程」打开弹窗。
- 新建课程至少包含课程名称、描述、创建时间、更新时间。
- 课程保存后出现在左侧课程列表。
- 选择课程后，中栏标题从「未选择课程」变为课程名称。
- 删除课程前必须二次确认。
- 无课程时继续显示「暂无课程」空状态。

验收标准：

- 新建课程后刷新应用仍可看到课程。
- 课程列表不出现任何演示或虚拟课程。
- 未选择课程时，思维导图和详情栏保持空状态。

实现状态：

- 已实现新建、选择、搜索、重命名、删除课程。
- 已实现 Notion 式左侧树状分区的新建、重命名、折叠、删除，以及课程在分区和「未分区」之间移动。
- 已接入 Electron 主进程 MySQL 数据层，课程分区写入 `knowledge_sections`，课程数据写入专用表 `course_management_courses`。
- 课程与分区表连接、建库、建表规则详见 `docs/功能规划/MySQL课程管理表.md`。
- 已支持复制本地定位路径，供外部 Codex 快速找到知识库对应数据。
- 后续仍需继续收口课程删除时的节点、边、文档和资产清理策略。

### 4.2 本地数据持久化

目标：先实现离线可用的本地数据层，为后续同步或数据库迁移留接口。

需求：

- 保存课程、节点、节点关系、文档内容、最近打开记录。
- 数据模型必须区分课程、图谱节点、文档三类实体。
- 所有写入操作要有失败提示，不允许静默失败。
- 数据文件或数据库位置需要固定，并在文档中说明。

建议模型：

- Course：课程。
- MindMapNode：思维导图节点。
- MindMapEdge：节点关系。
- Document：节点关联文档。
- AppSettings：用户偏好和布局状态。

验收标准：

- 关闭应用再打开，数据不丢失。
- 数据为空时展示真实空状态。
- 损坏或读取失败时给出中文错误提示。

### 4.3 思维导图编辑器

目标：中栏成为课程知识图谱的核心编辑区域。

需求：

- 选择课程后加载该课程的思维导图。
- 支持新增节点、编辑节点标题、删除节点。
- 支持建立父子关系或连线关系。
- 支持拖拽节点调整布局。
- 支持选中节点，并把节点详情同步到右栏。
- 支持撤销、重做、快捷键和自动保存。

验收标准：

- 新建节点后能保存并恢复。
- 选中节点时右栏展示对应标题和文档摘要。
- 删除节点会同步删除或处理关联边，并有确认提示。

实现状态：

- 已接入 `simple-mind-map`，导图是知识库的主编辑面。
- 已支持新增、删除、编辑、撤销、重做、主题色、导出、目录同步和自动保存。
- 文字排版工具已从顶部工具栏迁移到选中文字后的右键浮层。
- 分支排版工具改为快捷键驱动，快捷键设置在设置页维护。
- 已补入笔记、标签、链接、图片、优先级、进度、折叠等 XMind 类主题元素入口。
- 后续重点是继续补齐 XMind 的结构样式、概要、边界、关系线、标记、贴纸、查找替换和更完整的键盘工作流。

### 4.4 节点详情与文档

目标：右栏用于展示和编辑当前节点的学习资料。

需求：

- 选中节点后展示节点名称、备注、关联文档。
- 支持为节点创建或编辑文档。
- 支持 Markdown 基础编辑能力。
- 支持文档自动保存和保存状态提示。
- 支持节点无文档时的空状态和创建入口。

验收标准：

- 切换节点时右栏内容同步变化。
- 编辑文档后重新打开应用仍能恢复。
- 未选中节点时不展示上一次节点的旧内容。

实现状态：

- 已接入 `@hufe921/canvas-editor` 作为节点关联文档编辑器。
- 已支持导图和文档双模式切换、基础文字排版、分页阅读、上一页/下一页、清空和导入。
- 文档快照与导图节点通过稳定节点 id 绑定，不把大文档内容塞进导图节点记录。
- 已修正文档页面横向宽度和并排空白页问题，窗口或侧栏尺寸变化后会重建编辑器尺寸。
- 后续重点是继续补齐 Word 类能力，包括标题样式、列表、表格、图片、链接、查找替换、批注、目录、附件和导出。

### 4.5 搜索

目标：搜索框从静态展示变为可用检索入口。

需求：

- 支持按课程名称搜索。
- 选择课程后支持按节点标题搜索。
- 后续支持文档正文检索。
- 搜索结果为空时展示中文空状态。
- 搜索输入清空后恢复默认列表。

验收标准：

- 输入关键字后列表实时过滤。
- 搜索不会生成虚拟结果。
- 中文、英文、数字关键词均可检索。

### 4.6 工作台工具栏

目标：中栏右上角工具按钮具备明确功能。

需求：

- 思维导图按钮：切换或聚焦图谱视图。
- 文档视图按钮：切换到当前节点文档视图。
- 更多按钮：打开菜单，包含导入、导出、课程设置、删除课程。
- 未选择课程时工具栏保持禁用。
- 选择课程后按可用条件启用按钮。

验收标准：

- 禁用状态下按钮不可触发。
- 可用状态下点击有明确反馈。
- 更多菜单不会遮挡窗口边界。

实现状态：

- 知识库工作台已接入导图/文档切换、导入、导出、保存、目录、折叠等核心入口。
- 思维导图顶部工具栏只保留结构和通用操作，文字排版进入右键浮层，分支排版进入快捷键体系。
- 管理类功能沉到设置页，避免主工作台出现工程化控制台感。

### 4.7 详情栏折叠

目标：右栏可以被收起，为思维导图编辑留出空间。

需求：

- 点击「收起详情」隐藏右栏。
- 收起后中栏自动占用更多宽度。
- 提供恢复详情栏的入口。
- 记住用户上一次的详情栏展开状态。

验收标准：

- 折叠和展开不导致内容错位。
- 小屏幕下仍符合响应式布局。
- 重启应用后恢复上一次布局偏好。

实现状态：

- 左侧知识库栏和右侧目录栏均已支持折叠。
- 折叠只改变布局列宽和可见内容，不卸载当前中间工作台。
- 已修复左栏折叠时把中间导图或文档一起收掉的问题。

### 4.8 导入导出

目标：为课程资料迁移和备份提供能力。

需求：

- 支持导出单个课程为本地文件。
- 支持导入课程文件并合并到课程列表。
- 导入前校验文件格式。
- 导入失败时展示中文错误信息。

验收标准：

- 导出的课程可以在当前应用重新导入。
- 导入不会覆盖已有课程，除非用户明确确认。
- 导入后的节点和文档关系完整。

## 5. 非功能要求

- 所有界面文案必须为中文。
- 不允许使用虚拟课程、虚拟节点、虚拟文档作为默认数据。
- 未完成的功能入口必须禁用或隐藏。
- 桌面窗口必须使用原生窗口框架，关闭、最小化、最大化可用。
- 应用图标固定使用 `build/icon.ico`，后续打包不得回退默认 Electron 图标。
- 页面在 1080px 最小宽度下不得出现文字互相遮挡。
- 错误、空状态、保存状态必须有清晰中文反馈。
- MCP、更新、日志、诊断等管理能力必须放在设置或管理入口内，不压迫知识库主工作台。
- UI 必须保持商业级、简约导向，不展示实现细节、调试字段、变量名、接口名、技术栈痕迹或演示口号。

## 5.1 MCP 外部接入要求

目标：让 Codex、Claude Code 或其他设备可以通过 MCP 管理 AIstudy 知识库。

已实现范围：

- 读取：读取知识库列表、读取指定课程导图、搜索导图节点、读取文档内容。
- 编辑：编辑导图节点、追加节点、更新节点内容，继续扩展排版和文档编辑工具。
- 管控：健康检测、知识库定位、接入配置生成、权限开关、内网访问开关。
- 检测：本机和外部调用可记录状态，调用中亮灯，结束后熄灯，并可在设置中开启监控显示。
- 内网访问：检测 Tailscale 状态，启动本机服务，显示 MCP URL、API URL、Authorization 三行。

后续要求：

- MCP 必须支持全库管理，不能依赖用户当前 UI 选中了哪个课程或导图。
- 读取、搜索、编辑都必须支持明确指定课程、节点、文档或本地定位路径。
- 外部可编辑权限必须细分，读取、导图编辑、文档编辑、管理操作分别开关。
- 新设备接入时，界面只展示必要操作顺序和连接信息，不展示内部实现字段。

## 6. 推荐开发顺序

1. 继续补齐 XMind 对齐能力。
2. 继续补齐 Word 类文档编辑能力。
3. 完善 MCP 全库读取、搜索、导图编辑、文档编辑和权限模型。
4. 完善导入导出、资产、备份、恢复和迁移。
5. 完善搜索，从课程/节点扩展到文档全文。
6. 完善设置页，把管理类功能统一收口。
7. 每次功能变更后执行真实构建或打包验证。
8. 发布前验证安装包、桌面快捷方式、数据库迁移和旧数据保留。

## 7. 阶段验收清单

- 应用启动后没有乱码。
- 左侧导航只显示当前页面入口。
- 页面没有虚拟数据。
- 未完成按钮不会触发任何动作。
- 已接入功能的按钮可点击、有反馈、能持久化。
- 关闭应用后重新打开，课程、节点、文档、布局状态不丢失。
- 关闭应用后重新打开，MCP 内网访问和权限设置按用户配置恢复。
- `npm run build` 通过。
- 打包后 exe 能打开和正常关闭。
