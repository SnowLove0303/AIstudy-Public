# 思维导图目录层级约束

## 固定关系

- 导图节点树是唯一数据源，目录不得单独存储、单独编辑或单独排序。
- 导图根节点固定映射为目录一级。
- 导图 `children` 固定映射为目录子级。
- 目录的 `path` 使用 `1.1.2` 这类稳定层级路径表达父子关系。
- 目录的 `parentPath` 必须等于父节点的 `path`，根节点为 `null`。
- 目录的 `level` 从 `0` 开始，等于导图节点在树里的深度。
- 目录的 `order` 从 `1` 开始，等于同级节点在 `children` 里的顺序。

## 代码入口

- 目录只能通过 `buildMindMapOutline(root)` 生成。
- 导图节点树必须通过 `normalizeMindMapTree(root)` 归一化后进入保存、渲染或目录生成流程。
- 目录项类型必须使用 `MindMapOutlineItem`，不得新建平行目录类型绕过固定字段。

## 禁止事项

- 禁止把目录作为独立数据库表或独立 localStorage 结构保存。
- 禁止按标题匹配目录父子关系，标题允许重复。
- 禁止在目录面板里直接修改目录层级，层级变化必须来自导图节点树变化。
- 禁止新增功能时绕过 `path / parentPath / level / order` 字段推断层级。
